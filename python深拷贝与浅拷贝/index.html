#Python

Summary

  1. 可变数据类型与不可变数据类型
    1. 不可变数据类型：当我们对字符串、数值型、布尔值的数据改变变量名，并不会影响到数据在内存中的位置。
    2. 可变数据类型：虽然是相同的数据，但是变量名字不同，内存中仍然会开辟新的内存地址来进行存放相同的数据，

  2. 浅拷贝与深拷贝

    1. 浅拷贝只是拷贝数据的第一层，不会拷贝子对象。
    2. 深拷贝不仅会拷贝数据的第一层，还会拷贝数据的元素。
    3. 知道如何解释这个？吗？

Python对象

在python中，一切皆可对象。换句话说，我们在pyhton中构造的任何数据类型都可以是一个对象，不管是数字，字符串，字典等常见数据类型，或者说函数，甚至是我们导入的模块。

所有python的对象拥有三个属性：

  • 身份（数据地址，表示身份）
  • 类型
  • 值

甚至，type也是对象。

python中，万物皆可对象

数据类型

数据类型分为两种可变数据类型与不可变数据类型。

  • 不可变数据类型： Int， string，Bool, Tuple,数值，字符串，布尔值, 所谓不可变数据类型是指对象的身份（地址）和值都不可变。新创建的对象被关联到原来的变量名，旧的对象被丢弃，垃圾回收器会在适合的时候回收这些对象。
  • 可变数据类型：List, Dict, Set， 所谓可变数据类型是指对象的值可变，但是身份（地址）是不可变的。

深拷贝

深拷贝不同于浅拷贝的是：深拷贝会拷贝所有的可变数据类型，包含嵌套的数据中的可变数据。

深拷贝是变量对应的值复制到新的内存地址中，而不是复制数据对应的内存地址。

我们可以得出结论：

  • 深拷贝对最外层数据是只拷贝数据，会开辟新的内存地址来存放数据。

换句话说，当不存在嵌套的时候，对于可变数据类型，深拷贝和浅拷贝本质上是一样，但是如果存在嵌套呢？
